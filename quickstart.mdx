<Note>
  This quickstart guide will have you running the E-commerce API locally in under 5 minutes.
</Note>

## Prerequisites

Before you begin, ensure you have the following installed:
- Java 17 or higher
- Maven 3.6+
- Docker and Docker Compose
- Git

<Steps>
  <Step title="Clone the repository">
    Clone the E-commerce backend repository to your local machine:

    ```bash
    git clone <repository-url>
    cd ecommerce-backend
    ```
  </Step>

  <Step title="Start Keycloak with Docker Compose">
    The API uses Keycloak for authentication. Start it using Docker Compose:

    ```bash
    docker-compose up -d
    ```

    This will start Keycloak on port 8080 with:
    - Admin credentials: `admin` / `admin`
    - Pre-configured realm: `ecommerce`
    - Client ID: `ecommerce-module`
    - Client secret: `ecommerce-secret`

    <Note>
      Wait for Keycloak to be fully ready (about 30 seconds). The healthcheck will verify the realm is loaded.
    </Note>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file or export the required environment variables:

    <CodeGroup>
    ```bash .env file
    # Server
    SERVER_PORT=8081

    # Database
    SPRING_DATASOURCE_URL=localhost:3306/ecommerce
    SPRING_DATASOURCE_USERNAME=your_username
    SPRING_DATASOURCE_PASSWORD=your_password

    # JWT
    JWT_SECRET=your-secret-key-min-256-bits

    # Keycloak
    KEYCLOAK_CLIENT_SECRET=ecommerce-secret

    # Email (optional for quickstart)
    SPRING_MAIL_USERNAME=your-email@gmail.com
    SPRING_MAIL_PASSWORD=your-app-password

    # Kafka
    KAFKA_BOOTSTRAP=localhost:9092
    KAFKA_USERNAME=your-kafka-user
    KAFKA_PASSWORD=your-kafka-password
    ```

    ```bash Export commands
    export SERVER_PORT=8081
    export SPRING_DATASOURCE_URL=localhost:3306/ecommerce
    export SPRING_DATASOURCE_USERNAME=your_username
    export SPRING_DATASOURCE_PASSWORD=your_password
    export JWT_SECRET=your-secret-key-min-256-bits
    export KEYCLOAK_CLIENT_SECRET=ecommerce-secret
    ```
    </CodeGroup>

    <Warning>
      For a quick local test, you can use MySQL running on localhost. See the [Installation guide](/installation) for detailed database setup.
    </Warning>
  </Step>

  <Step title="Build and run the application">
    Build and start the Spring Boot application:

    ```bash
    mvn clean install
    mvn spring-boot:run
    ```

    The API will start on `http://localhost:8081` (or your configured port).

    You should see output indicating the application has started:

    ```bash
    INFO 12345 --- [main] a.e.u.ecommerce.EcommerceApplication : Started EcommerceApplication in 5.234 seconds
    ```
  </Step>

  <Step title="Test the API">
    Test that the API is running by making a request to register a new user:

    ```bash cURL
    curl -X POST http://localhost:8081/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "email": "test@example.com",
        "password": "securePassword123",
        "name": "Test",
        "lastname": "User"
      }'
    ```

    Then login to get an authentication token:

    ```bash cURL
    curl -X POST http://localhost:8081/api/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "test@example.com",
        "password": "securePassword123"
      }'
    ```

    You should receive a response with a JWT bearer token:

    ```json Response
    {
      "success": true,
      "bearer_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": 1,
        "name": "Test",
        "lastname": "User",
        "email": "test@example.com",
        "addresses": []
      }
    }
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Keycloak is not starting">
    - Ensure port 8080 is not already in use
    - Check Docker logs: `docker-compose logs keycloak`
    - Wait longer for the healthcheck to pass (can take up to 1 minute)
  </Accordion>

  <Accordion title="Database connection errors">
    - Verify MySQL is running and accessible
    - Check your database credentials in environment variables
    - Ensure the database `ecommerce` exists
    - Verify the database URL format: `localhost:3306/ecommerce`
  </Accordion>

  <Accordion title="JWT token errors">
    - Ensure JWT_SECRET is at least 256 bits (32 characters)
    - Check that the secret doesn't contain special characters that need escaping
  </Accordion>

  <Accordion title="Port 8081 already in use">
    - Change the SERVER_PORT environment variable to a different port
    - Kill any existing process using port 8081
  </Accordion>
</AccordionGroup>
