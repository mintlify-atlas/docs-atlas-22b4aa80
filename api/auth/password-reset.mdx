The password reset functionality is handled through three separate endpoints that work together to provide a secure password reset flow.

## Endpoints Overview

1. **Request Password Reset** - Initiates the reset process and sends a token via email
2. **Validate Token** - Verifies the reset token is valid
3. **Change Password** - Completes the reset with a new password

---

## Request Password Reset

<api>POST /password/request</api>

Initiates the password reset process by sending a recovery code to the user's email.

### Authentication

This endpoint does not require authentication.

### Request Body

<ParamField body="email" type="string" required>
  The email address of the account to reset
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message confirming the email was sent
</ResponseField>

<ResponseField name="error" type="string">
  Error message if the email doesn't belong to any account
</ResponseField>

### Response Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Reset email sent successfully |
| 404 | Email not found in system |

### Example

<CodeGroup>

```bash cURL
curl -X POST https://api.example.com/password/request \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com"
  }'
```

```java Java
@PostMapping("/request")
public ResponseEntity<?> requestPasswordReset(@RequestBody Map<String, String> body) {
    String email = body.get("email");
    try {
        passwordResetService.requestPasswordReset(email);
        return ResponseEntity.ok(Map.of("message", "Se ha enviado un correo con el código de recuperación"));
    } catch (RuntimeException e) {
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(Map.of("error", "El email no pertenece a ninguna cuenta"));
    }
}
```

```json Response (Success - 200)
{
  "message": "Se ha enviado un correo con el código de recuperación"
}
```

```json Response (Not Found - 404)
{
  "error": "El email no pertenece a ninguna cuenta"
}
```

</CodeGroup>

## Validate Token

<api>POST /password/validate</api>

Validates that the reset token is correct before allowing password change.

### Authentication

This endpoint does not require authentication.

### Request Body

<ParamField body="email" type="string" required>
  The email address of the account
</ParamField>

<ParamField body="token" type="string" required>
  The reset token received via email
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message if token is valid
</ResponseField>

<ResponseField name="error" type="string">
  Error message if token is invalid
</ResponseField>

### Response Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Token is valid |
| 400 | Token is invalid or incorrect |

### Example

<CodeGroup>

```bash cURL
curl -X POST https://api.example.com/password/validate \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "token": "123456"
  }'
```

```java Java
@PostMapping("/validate")
public ResponseEntity<?> validateToken(@RequestBody Map<String, String> body) {
    String email = body.get("email");
    String token = body.get("token");
    boolean valid = passwordResetService.validateToken(email, token);
    if (valid) {
        return ResponseEntity.ok(Map.of("message", "Token válido"));
    } else {
        return ResponseEntity.badRequest().body(Map.of("error", "El token enviado no es correcto"));
    }
}
```

```json Response (Valid - 200)
{
  "message": "Token válido"
}
```

```json Response (Invalid - 400)
{
  "error": "El token enviado no es correcto"
}
```

</CodeGroup>

## Change Password

<api>POST /password/change</api>

Completes the password reset by setting a new password.

### Authentication

This endpoint does not require authentication.

### Request Body

<ParamField body="email" type="string" required>
  The email address of the account
</ParamField>

<ParamField body="token" type="string" required>
  The valid reset token received via email
</ParamField>

<ParamField body="newPassword" type="string" required>
  The new password to set for the account
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message confirming password was changed
</ResponseField>

<ResponseField name="error" type="string">
  Error message if token is invalid or expired
</ResponseField>

### Response Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Password changed successfully |
| 400 | Invalid token or request failed |

<Warning>
  The token must be valid and not expired. Each token can typically only be used once for security reasons.
</Warning>

### Example

<CodeGroup>

```bash cURL
curl -X POST https://api.example.com/password/change \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "token": "123456",
    "newPassword": "newSecurePassword123"
  }'
```

```java Java
@PostMapping("/change")
public ResponseEntity<?> changePassword(@RequestBody Map<String, String> body) {
    String email = body.get("email");
    String token = body.get("token");
    String newPassword = body.get("newPassword");
    try {
        passwordResetService.changePassword(email, token, newPassword);
        return ResponseEntity.ok(Map.of("message", "Contraseña cambiada exitosamente"));
    } catch (RuntimeException e) {
        return ResponseEntity.badRequest().body(Map.of("error", "El token enviado no es correcto"));
    }
}
```

```json Response (Success - 200)
{
  "message": "Contraseña cambiada exitosamente"
}
```

```json Response (Invalid Token - 400)
{
  "error": "El token enviado no es correcto"
}
```

</CodeGroup>

## Complete Password Reset Flow

<Note>
  The recommended flow for password reset:
  
  1. User requests password reset via `/password/request`
  2. System sends recovery token to user's email
  3. User enters token and validates it via `/password/validate` (optional but recommended)
  4. User submits new password with token via `/password/change`
  5. User can now log in with the new password
</Note>
