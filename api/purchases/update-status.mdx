---
title: "Update Purchase Status"
api: "POST /purchase/{id}/confirm/{addressId}"
description: "Confirm a pending purchase and set delivery address"
---

## Authentication

This endpoint requires Bearer token authentication. The user must have an active session.

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the purchase to confirm
</ParamField>

<ParamField path="addressId" type="integer">
  The delivery address ID (optional)
</ParamField>

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

## Response

<ResponseField name="id" type="integer">
  Purchase ID
</ResponseField>

<ResponseField name="status" type="string">
  Updated purchase status (CONFIRMED)
</ResponseField>

<ResponseField name="date" type="string">
  Purchase date (ISO 8601 format)
</ResponseField>

<ResponseField name="reservationTime" type="string">
  Reservation timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="direction" type="string">
  Delivery address description
</ResponseField>

<ResponseField name="cart" type="object">
  Associated cart object
</ResponseField>

## Business Logic

1. **Status Update**: Changes purchase status from PENDING to CONFIRMED
2. **Address Assignment**: If addressId is provided, fetches and assigns the address description
3. **Stock Confirmation**: Confirms product stock reservation via Kafka event
4. **Event Emission**: Sends "purchase confirmed" event with user and cart details

## Error Responses

- **401 Unauthorized**: User is not authenticated or session is inactive
- **400 Bad Request**: Invalid or non-existent addressId provided
- **200 OK with null body**: Purchase with specified ID does not exist

## Related Endpoints

- **DELETE /purchase/{id}**: Cancel a purchase and restore stock
- **GET /purchase/pending-cart**: Get the current pending cart
- **GET /purchase/{id}**: View purchase details

## Notes

- Purchase must be in PENDING status to be confirmed
- Product stock is permanently committed upon confirmation
- Confirmation triggers inventory updates via Kafka
- Address is optional but recommended for physical deliveries
- Returns 200 OK with null body if purchase doesn't exist
