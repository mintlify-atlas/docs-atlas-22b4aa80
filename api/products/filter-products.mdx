## Endpoint

```
POST /products/filter
```

## Description

Filters products based on price range, brand, and category criteria with sorting and pagination support. Only active products are returned.

## Authentication

No authentication required.

## Request Body

<ParamField body="priceMin" type="float">
  Minimum price filter (inclusive)
</ParamField>

<ParamField body="priceMax" type="float">
  Maximum price filter (inclusive)
</ParamField>

<ParamField body="brandCode" type="integer">
  Single brand code to filter by
</ParamField>

<ParamField body="brandCodes" type="array">
  Multiple brand codes to filter by (takes precedence over single brandCode)
</ParamField>

<ParamField body="categoryCode" type="integer">
  Single category code to filter by
</ParamField>

<ParamField body="categoryCodes" type="array">
  Multiple category codes to filter by (takes precedence over single categoryCode)
</ParamField>

<ParamField body="sortBy" type="string">
  Sort field: `"price"` or `"relevance"` (sorts by calification)
</ParamField>

<ParamField body="sortOrder" type="string">
  Sort direction: `"asc"` or `"desc"`
</ParamField>

<ParamField body="page" type="integer" default="0">
  Page number for pagination (0-based)
</ParamField>

<ParamField body="size" type="integer" default="20">
  Number of items per page
</ParamField>

## Filtering Logic

### Active Products Only
Only products with `active=true` are included (ProductController.java:225)

### Price Range
- Products with null prices are excluded if price filters are specified
- `priceMin`: Filters products greater than or equal to minimum price (ProductController.java:229)
- `priceMax`: Filters products less than or equal to maximum price (ProductController.java:232)

### Brand Filtering
- `brandCodes` array takes precedence over single `brandCode` (ProductController.java:236)
- Products without a brand are excluded when brand filter is active
- Uses brand's `brandCode` field for matching (ProductController.java:237)

### Category Filtering
- `categoryCodes` array takes precedence over single `categoryCode` (ProductController.java:246)
- Products must have at least one matching category (ProductController.java:248)
- Uses category's `categoryCode` field for matching

## Sorting Options

### By Price
```json
{
  "sortBy": "price",
  "sortOrder": "asc"  // or "desc"
}
```
Sorts products by price field (ProductController.java:260-265)

### By Relevance
```json
{
  "sortBy": "relevance",
  "sortOrder": "desc"  // or "asc"
}
```
Sorts products by calification (rating) field (ProductController.java:266-271)

## Pagination

- Filtering is applied first to all products
- Sorting is applied to filtered results
- Pagination is applied last with manual slicing (ProductController.java:275-280)
- Default page: 0
- Default size: 20

## Implementation Notes

- All filtering is done in-memory after fetching all products
- Products with null brand/categories are excluded when those filters are active
- Multiple brand/category codes use OR logic (match any)
- Price and brand/category filters use AND logic (must match all specified filters)
